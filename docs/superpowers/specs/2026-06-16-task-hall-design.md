## 核心决策总览

| 决策 | 选择 |
|------|------|
| 匹配模式 | **混合**：加急推送 + 长线池子，超时重推 |
| 履约约束 | **质押扣点 + 凭证上传**：物流单号 + 内容链接 |
| 审核模式 | **无审核**：KOC 发链接即完成，事后评分驱动匹配权重 |
| 任务大厅 | 商家看进度不看人（匿名代号），KOC 看收益不看排名 |
| 通知渠道 | MVP 邮件（gmail-assistant），后续站内通知 |
| 意向系统 | **保留**，作为长线任务补充入口 |
| 经济模型 | 独立积分体系，≥15% 纯利，双方抽，**后续专题** |

## 一、任务生命周期状态机

```
商家发布任务
  │
  ▼
⚡ 待匹配 (pending)
  │ 平台匹配引擎自动推送
  ▼
📨 待接单 (assigned)
  │ KOC 接受 → 扣双方质押点
  │ KOC 拒绝/超时(12h) → 🔄 重新匹配(从 buffer 补位)
  ▼
📦 待发货 (accepted)
  │ 商家上传物流单号
  │ 超时 48h 未发货 → ⚠️ 商家违约：退 KOC 质押 + 扣商家点给 KOC
  ▼
🚚 已发货 (shipped)
  │ KOC 确认收货
  │ 超时 7 天未确认 → 平台自动确认
  ▼
✍️ 创作中 (creating)
  │ KOC 上传内容链接
  │ 超时 14 天未提交 → ⚠️ KOC 违约：退商家质押 + 扣 KOC 点给商家
  ▼
✅ 已完成 (completed)
  │ 释放双方质押 + 结算佣金点数
  ▼
⭐ 待互评 → 互评完成 → 归档
  │
  ▼  事后：平台拉取内容数据表现 → 更新 KOC 评分
```

### 超时规则

| 节点 | 超时 | 后果 |
|------|------|------|
| KOC 待接单 | 12 小时 | 自动重推下一个 KOC（从 buffer 补位） |
| 商家待发货 | 48 小时 | 商家违约，质押扣给 KOC |
| 已发货未确认 | 7 天 | 平台自动确认收货 |
| 创作中未提交 | 14 天 | KOC 违约，质押扣给商家 |

### 任务双轨

| 类型 | 加急 (urgent) | 长线 (long_term) |
|------|:--:|:--:|
| 匹配方式 | 平台主动推送 | 挂在产品池，KOC 自己发现 |
| 接单时限 | 12h | 无强制时限 |
| 超时重推 | ✅ | ❌ |
| KOC 表达意向 | 不需要 | ✅ 可点"感兴趣" |
| 出现位置 | 任务广场"加急"tab + 推送通知 | 产品池 + 任务广场"长线"tab |

## 二、评分体系 + 匹配权重

### KOC 评分维度

| 维度 | 来源 | 权重 | 更新频率 |
|------|------|:--:|------|
| 🎯 内容真实性 | AI 初评 + 商家反馈 | 25% | 每次履约后 |
| 🏷️ 品类匹配度 | AI 初评 + 完成品类数 | 25% | 每次履约后 |
| 📊 数据表现 | 事后拉取（播放/互动/转化） | 30% | 每周 |
| ⭐ 商家互评 | 1-5 星 | 10% | 每次履约后 |
| 🔥 活跃度 | 接单速度 + 完成率 + 是否 Ghosted | 10% | 实时 |

### 商家评分维度

| 维度 | 来源 | 权重 |
|------|------|:--:|
| 📦 发货时效 | 系统记录 | 25% |
| 🏷️ 产品真实性 | KOC 反馈 + 退货率 | 20% |
| ⭐ KOC 互评 | 1-5 星 | 20% |
| 💰 结算速度 | 系统记录 | 15% |
| 🛡️ 诚信度 | 综合（见下方） | 20% |

### 商家诚信度 (trust_score，0-100，默认 100)

**扣分规则：**

| 行为 | 扣分 | 说明 |
|------|:--:|------|
| 超时未发货（48h） | -20 | 单次违约 |
| 发货后 KOC 投诉货不对板 | -25 | 经 Admin 确认 |
| 7 天内被 2+ 个 KOC 拉黑 | -15 | 累计触发 |
| 协商退款不结算 | -5 | 轻微 |
| 账号被平台警告 | -30 | Admin 手动 |

**奖励规则：**

| 行为 | 加分 | 说明 |
|------|:--:|------|
| 24h 内发货（连续 5 单） | +5 | 快速响应 |
| 累计完成 20 单无争议 | +5 | 长期良好记录 |
| KOC 平均评分 ≥ 4.5 | +5 | 每 10 单检查一次 |

**效果：**
- 诚信度 < 60：任务在广场展示时标记 🛡️ 低信，KOC 可见
- 诚信度 < 40：暂停新任务发布权限，已有任务继续
- 诚信度 < 20：冻结账号，Admin 介入
- 诚信度恢复：每完成 3 个无争议任务自动 +5

### 匹配权重公式

```
匹配分 = ( KOC分 × 0.55 + 商家分 × 0.45 )
       × 品类匹配加成 (×1.0 ~ 1.5)
       × 新人加成 (前 5 单 ×1.3)
       × 诚信系数 (诚信度/100，即 0.4 ~ 1.0 线性)
```

**新人加成**：新 KOC 前 5 个任务有 1.3 倍权重，确保不会被高等级垄断。

### 劣质惩罚

KOC 连续发布质量差的后果：
- 数据表现分下降 → 匹配权重降低
- 匹配到的任务变少/佣金变低
- 不封号不拉黑，纯评分驱动

## 三、匹配引擎

取代当前 Admin 手动匹配的自动化：

```
商家发布任务（需 N 个 KOC, 品类=美妆, 佣金=30点, 加急）
    │
    ▼
┌──────────────────────────────────────┐
│  匹配引擎                             │
│  1. 筛选：品类匹配 + 未拉黑 + 活跃     │
│     + 商家诚信度 ≥ 40（低于40不参与） │
│  2. 排除：已有冲突任务的 KOC           │
│  3. 排序：匹配权重分降序               │
│  4. 加新人因子：前 5 单 ×1.3          │
│  5. 取 Top (N + buffer)，buffer=3     │
│  6. 分批推送：每次推 3 个，间隔 2h     │
└──────────────┬───────────────────────┘
               │
               ▼
         通知 KOC → 12h 内接单
         拒绝/超时 → 自动从 buffer 补位
         若 buffer 耗尽 → 扩大筛选重新匹配
```

### 重推策略

- KOC 拒绝或 12h 超时 → 自动从 buffer 取下一个推送
- 每拒绝一次，该 KOC 与该商家的匹配权重 -10%（防反复推送）
- buffer 耗尽 → 重新跑匹配引擎

## 四、任务广场排序算法

KOC 看到的任务列表排序：

```
排序分 = 发布时间衰减分 × 0.30
       + 紧急度分(加急=100, 普通=0) × 0.25
       + 条件匹配分(品类×等级) × 0.25
       + 佣金分(点数归一化) × 0.20
```

即：新的在前 → 加急提权 → 匹配度高的推 → 点数多的排前面。

## 七、数据模型变更

### `Merchant` 新增字段

```python
trust_score: int = 100          # 诚信度，0-100，默认 100
total_tasks_completed: int = 0  # 累计完成任务数（恢复诚信度用）
total_tasks_disputed: int = 0   # 累计争议数
```

### `KocTask` 改为批量模型

```python
class KocTask(BaseModel):
    id: str
    merchant_id: str
    product_id: str
    product_name: str

    # 新增：任务类型
    task_type: str = "urgent"      # urgent | long_term
    task_status: str = "pending"   # pending|assigned|accepted|shipped|creating|completed

    # 新增：批量 KOC 子任务
    koc_required: int = 1          # 需要几个 KOC
    koc_slots: list = []           # [{koc_id, status, assigned_at, accepted_at, 
                                   #   tracking_number, content_urls, content_data, 
                                   #   pledge_paid, commission_paid}]

    # 新增：质押 + 佣金
    pledge_merchant: int = 0       # 商家单次质押点
    pledge_koc: int = 0            # KOC 单次质押点
    commission: int = 0            # KOC 完成佣金

    # 新增：物流
    tracking_number: str = ""

    # 新增：内容
    content_urls: list = []

    # 保留
    due_at: str = ""
    created_at: str = ""
```

### 新增 `TaskHallQuery` 视图参数

```python
class TaskHallQuery(BaseModel):

    category: str = ""            # 品类筛选
```

## 八、与现有系统的关系

| 现有模块 | 处理方式 |
|------|------|
| 意向系统 (Interest) | **保留**，长线任务补充入口；KOC 浏览产品点"有意向" → 商家看到后可以主动发起匹配 |
| KOC 申请 → AI 评分 | **保留**，不变 |
| Admin 审核面板 | **去重**，匹配自动化后 Admin 只做异常仲裁 |
| Admin 手动创建任务 | **废弃**，改为商家自主发布 |
| 折扣码 / 裂变 | **保留**，不变 |
| 黑名单 | **保留**，加入匹配引擎的排除逻辑 |
| 互评 | **保留**，作为评分维度 |
| Cron 扫描 | **增强**，加超时检测 + 自动重推 + 数据拉取 |
